    **Extend visual-qa-hook.js** -- AFTER the screenshot capture step (do NOT rewrite the existing filtering/capture logic), add:

    1. **Baseline lookup**: Determine baseline name from the screenshot context. Use `jarvis-dashboard.png` as the baseline name per D-14. Full path: `~/.visual-qa/baselines/jarvis-dashboard.png`.

    2. **First-run handling**: If no baseline exists for this name, copy the current screenshot AS the baseline. Output advisory: "Visual QA: No baseline found -- saving current screenshot as baseline at {path}". Write no regression state. Exit.

    3. **Pixel comparison per D-08, D-09, D-10**:
       ```javascript
       const { PNG } = require('pngjs');
       const pixelmatch = require('pixelmatch');

       const baseline = PNG.sync.read(fs.readFileSync(baselinePath));
       const current = PNG.sync.read(fs.readFileSync(screenshotPath));

       // Images must be same dimensions -- if not, resize current to match baseline
       // or treat as 100% diff (layout changed dramatically)
       if (baseline.width !== current.width || baseline.height !== current.height) {
         // Treat dimension mismatch as full regression
         diffPercent = 1.0;
       } else {
         const diff = new PNG({ width: baseline.width, height: baseline.height });
         const numDiffPixels = pixelmatch(
           baseline.data, current.data, diff.data,
           baseline.width, baseline.height,
           { threshold: 0.1 } // per-pixel sensitivity (not the 5% overall threshold)
         );
         const totalPixels = baseline.width * baseline.height;
         diffPercent = numDiffPixels / totalPixels;

         // Save diff image per Claude's discretion
         fs.writeFileSync(diffImagePath, PNG.sync.write(diff));
       }
       ```

    4. **Threshold check per D-09**: Compare `diffPercent` against `config.threshold` (0.05 = 5%).

    5. **State file for commit guard**: Write regression state to `/tmp/visual-qa-state.json`:
       ```json
       {
         "has_regression": true,
         "baseline": "jarvis-dashboard.png",
         "diff_percent": 0.073,
         "diff_image": "/Users/claw2501/.visual-qa/diffs/jarvis-dashboard-diff.png",
         "screenshot": "/Users/claw2501/.visual-qa/screenshots/jarvis-dashboard-latest.png",
         "timestamp": "2026-04-03T..."
       }
       ```
       When diff is BELOW threshold, write `{ "has_regression": false, "timestamp": "..." }`.

    6. **Advisory output**: When regression detected (>5%), output via `hookSpecificOutput.additionalContext`:
       ```
       VISUAL REGRESSION DETECTED: jarvis-dashboard.png has {X.X}% pixel difference (threshold: 5%).
       Diff image: {diff_path}
       To update baseline: bash scripts/update-visual-baseline.sh
       WARNING: git commit will be BLOCKED until baseline is updated or regression resolves.
       ```
       When no regression, output: "Visual QA: {X.X}% diff (within 5% threshold) -- OK"

    **Create scripts/update-visual-baseline.sh per D-12:**
    ```bash
    #!/bin/bash
    # Update visual QA baselines from latest screenshots
    BASELINES_DIR="$HOME/.visual-qa/baselines"
    SCREENSHOTS_DIR="$HOME/.visual-qa/screenshots"

    # Copy latest screenshot as new baseline
    latest=$(ls -t "$SCREENSHOTS_DIR"/jarvis-dashboard-*.png 2>/dev/null | head -1)
    if [ -z "$latest" ]; then
      echo "No screenshots found. Run a JARVIS frontend edit to trigger a screenshot first."
      exit 1
    fi

    cp "$latest" "$BASELINES_DIR/jarvis-dashboard.png"
    echo "Baseline updated: $BASELINES_DIR/jarvis-dashboard.png"
    echo "Source: $latest"

    # Clear regression state
    echo '{"has_regression":false,"timestamp":"'$(date -u +%Y-%m-%dT%H:%M:%SZ)'"}' > /tmp/visual-qa-state.json
    echo "Regression state cleared. Commits are unblocked."
    ```
    Make executable: `chmod +x scripts/update-visual-baseline.sh`
