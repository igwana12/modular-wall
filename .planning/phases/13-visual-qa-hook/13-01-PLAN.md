    Create `/Users/claw2501/.claude/hooks/visual-qa-hook.js` as a PostToolUse hook:

    1. **Stdin parsing**: Read JSON from stdin with a 10s timeout guard (same pattern as obsidian-archive.js). Parse `tool_name` and `tool_input.file_path`.

    2. **Filter**: Only process `Write`, `Edit`, `MultiEdit` tool calls. Exit(0) for all others.

    3. **Path check per D-01**: Check if `file_path` starts with `/Volumes/Extreme Pro/ACTIVE/jarvis/frontend/`. Exit(0) if not.

    4. **Extension check per D-02**: Check if file_path ends with `.tsx`, `.css`, or `.html`. Exit(0) if not.

    5. **JARVIS running check per D-06**: Use `http.get` to check if JARVIS is running at the configured URL (from config.json at `~/.visual-qa/config.json`). If not responding within 2s, output advisory warning via `hookSpecificOutput.additionalContext`: "Visual QA: JARVIS not running at {url} -- screenshot skipped". Then exit(0).

    6. **Screenshot capture per D-05 and IMPORTANT TECHNICAL NOTE**: Since the hook runs as a standalone Node.js script (NOT inside Claude Code's tool context), it CANNOT use MCP tools. Use Playwright's Node.js API directly:
       ```javascript
       const { chromium } = require('playwright');
       ```
       - Launch headless chromium
       - Navigate to JARVIS URL
       - Wait 2 seconds for rendering (JARVIS has animations)
       - Take full-page screenshot
       - Save to `.visual-qa/screenshots/jarvis-dashboard-{timestamp}.png`
       - Close browser

       If Playwright is not installed as a Node.js module, fall back to spawning `npx playwright screenshot {url} {output_path}` as a child process.

    7. **Output**: On successful screenshot, output advisory via `hookSpecificOutput.additionalContext`: "Visual QA: Screenshot captured at {path}". On failure, warn but never block (exit(0) on any error).

    8. **Error handling**: Wrap everything in try/catch. Any error = exit(0) silently. Never crash, never block the tool execution. Log errors to `/tmp/visual-qa-hook.log` for debugging.

    CRITICAL: The hook must be fast. Use a 15s overall timeout. If screenshot takes longer, abort and warn.
    CRITICAL: Use `require('playwright')` first, and if that fails (module not found), fall back to `child_process.execSync('npx playwright screenshot ...')`.
