## Decisions Made

1. **Plotly CDN loaded after ECharts CDN** -- Both chart libraries coexist. ECharts handles 2D charts, Plotly handles 3D. chart_library field in WS messages determines routing.

2. **Widget overlay via CSS class (not reparenting)** -- Adding/removing widget-overlay-mode class is simpler and avoids DOM mutation side effects that could break DataCardRenderer state.

3. **Gradient syncDuration via speed override** -- Rather than modifying the requestAnimationFrame loop, syncDuration sets a higher _transitionSpeed (1000/durationMs) and resets after completion. Non-intrusive to the existing update() call from renderLayers().

4. **S9 stagger via CSS custom properties** -- Using --s9-angle, --s9-tz, --s9-ty custom properties in the @keyframes animation avoids conflicts with the inline transform that positions characters in the 3D ring.
