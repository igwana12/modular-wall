# Phase 07.4 Plan 02: Spatial Layout Manager + Auto-Animate Morphing Summary

Panels auto-position based on active neighbors (split/dodge/pip) with smooth CSS transitions, and consecutive same-module storyboard segments morph content in place instead of hide/re-show cycling.

## Tasks Completed

### Task 1: Spatial layout manager + position flow in VisualStageController and MediaPresenter
**Commit:** `03a2482`

Added `_resolvePosition(key, msg)` to VisualStageController that inspects active panel set, layer types, and layout hints to assign `data-position` attributes and layout CSS classes. When a beside-layer panel (chart, map, graph) is active on the left, new panels auto-dodge to the right. Layout hints from ScriptSegment (`split`, `overlay`, `pip`) produce `viz-layout-split`, `viz-layout-overlay`, `viz-layout-pip` CSS classes with correct positioning. Added `_recalcPositions()` called from both `_fadeOut` and `_hidePanel` to recenter remaining panels when one departs. Added `updatePosition(position)` method to MediaPresenter for external spatial coordination. CSS layout classes in index.html include smooth 0.4s transition for position changes.

### Task 2: Auto-animate morph detection in StoryboardScheduler
**Commit:** `6f4cbdb`

Added `_lastFiredSegment` tracking to StoryboardScheduler constructor (reset on stop). Modified `_fireSegment()` to compare current segment module with previous via `_isSamePanel()` normalizer (strips `_show`/`_hide`/`_slide`/`_update` suffixes). When consecutive segments target the same panel, sets `_morph = true` and `_morphTransition = true` on segment data. Added morph shortcut in VisualStageController.handleMessage that checks `msg._morph && this._active.has(key)` -- when true, calls panel.handleMessage directly (or map.flyTo for map panels) bypassing the queue/drain/fadeOut/fadeIn cycle entirely. ECharts auto-animates via setOption, media uses existing crossfade path.

## Decisions Made

1. **Split layout assigns left to existing, right to newcomer** -- When two beside-layer panels are active simultaneously, the first panel keeps its position (or gets assigned left if it had none) and the new panel takes right. This avoids position thrashing.

2. **Morph path short-circuits before queue** -- The morph check happens at the top of handleMessage, before dedup/queue/drain logic. This gives morph updates zero latency -- no waiting for transition timers or queue drain.

3. **Panel suffix normalization for morph matching** -- `_isSamePanel()` strips `_show`, `_hide`, `_slide`, `_update` suffixes before comparing. This means `chart_show` followed by `chart_show` correctly detects as same panel, even if one segment uses `module: "chart"` and another uses `data.type: "chart_show"`.
