**ENGINE-02: Swappable personality protocols:**

The deity JSON configs in gods/ ARE the personality protocols. "Swappable" means: (a) the backend serves a list of available protocols, (b) devices can download a specific protocol config, (c) WebSocket clients can hot-swap deity mid-session without reconnecting, (d) new protocol files can be added to gods/ and picked up via reload.

1. In models.py, add `ProtocolInfo(BaseModel)` with fields: id (str), name (str), title (str), voice_id (str), reading_style (str), color_palette (list[str]), has_voice (bool). The `has_voice` field is `bool(voice_id)`.

2. In models.py, add `ProtocolListResponse(BaseModel)` with field: protocols (list[ProtocolInfo]), count (int).

3. In server.py, add `GET /api/protocols` endpoint that calls `list_deities()` and returns `ProtocolListResponse`. Map each DeityInfo to ProtocolInfo (add has_voice = bool(voice_id)).

4. In server.py, add `GET /api/protocols/{protocol_id}` endpoint that calls `load_deity(protocol_id)` and returns the full JSON config. This is what firmware downloads to switch personality.

5. In server.py, add `POST /api/protocols/reload` endpoint that calls `reload()` from deity_config and returns `{"reloaded": true, "count": len(list_deities())}`. This lets new JSON files be picked up without restarting the service.

6. In sphere_ws.py, handle a new message type `{"type": "swap_protocol", "protocol_id": "athena"}`. When received:
   - Load the new deity config via `load_deity(protocol_id)`
   - If found: update session `deity_id` variable, send `{"type": "status", "state": "protocol_swapped", "protocol_id": "athena"}`
   - If not found: send `{"type": "error", "message": "Unknown protocol: athena"}`
   - Do NOT require audio reset — next audio utterance uses new protocol
   - Log the swap

7. In deity_config.py, add `def get_protocol_ids() -> list[str]` that returns sorted list of all cached deity IDs. Useful for firmware to know what's available.
