<objective>
Build OTA (Over-The-Air) firmware update system: backend serves firmware binaries with version manifests, firmware client checks for updates and self-installs new versions over WiFi.

Purpose: OTA updates are essential for a product — users should never need to plug in a USB cable to update firmware after initial setup. This decouples firmware iteration speed from physical access to devices.

Output: Backend ota.py module with version manifest and binary serving endpoints, Arduino firmware module using ESP32 httpUpdate for self-update.
</objective>

**ENGINE-05: Firmware OTA client**

Create Arduino-compatible OTA module that the Phase 4 firmware will include. This is a standalone module — it compiles independently and gets #included into main firmware later.

1. Create `firmware/oracle-engine/` directory if it does not exist.

2. Create `firmware/oracle-engine/ota_update.h`:
```cpp
#ifndef OTA_UPDATE_H
#define OTA_UPDATE_H

#include <Arduino.h>

// Configuration
#define OTA_CHECK_INTERVAL_MS  (60 * 60 * 1000)  // Check every hour
#define OTA_SERVER_DEFAULT     "http://192.168.1.100:8300"  // Smithers IP

// Initialize OTA with server URL and current firmware version
void otaInit(const char* serverUrl, const char* currentVersion);

// Check for updates. Returns true if update was applied (device will reboot).
// Returns false if no update available or update failed.
bool otaCheckForUpdate();

// Get current firmware version string
const char* otaGetVersion();

// Get last OTA error message (empty string if no error)
const char* otaGetLastError();

#endif
```

3. Create `firmware/oracle-engine/ota_update.cpp`:
   - Include `<WiFi.h>`, `<HTTPClient.h>`, `<HTTPUpdate.h>`, `<ArduinoJson.h>`, `"ota_update.h"`
   - Store `_serverUrl`, `_currentVersion`, `_lastError` as static char arrays.
   - `otaInit()`: store server URL and version.
   - `otaCheckForUpdate()`:
     a. Build URL: `{serverUrl}/api/ota/check?version={currentVersion}`
     b. HTTPClient GET request with 10s timeout.
     c. Parse JSON response with ArduinoJson (StaticJsonDocument<512>).
     d. Check `update_available` field. If false, return false.
     e. If true, build download URL: `{serverUrl}/api/ota/download`
     f. Call `httpUpdate.update(wifiClient, downloadUrl)` where wifiClient is a WiFiClient instance.
     g. On `HTTP_UPDATE_OK`: log "Update success, rebooting", return true (ESP32 auto-reboots).
     h. On `HTTP_UPDATE_FAILED`: store error in `_lastError`, log error, return false.
     i. On `HTTP_UPDATE_NO_UPDATES`: return false.
   - `otaGetVersion()`: return `_currentVersion`.
   - `otaGetLastError()`: return `_lastError`.
   - Use `Serial.printf` for all logging with "[OTA]" prefix.
   - Note in comments: For production, add SHA256 verification of downloaded binary. Deferred to v2 for simplicity.

4. Create `firmware/oracle-engine/README.md` with:
   - Brief description: "Oracle Engine firmware modules for ESP32-S3"
   - OTA usage example showing otaInit + periodic otaCheckForUpdate in loop
   - Note that main.ino will be created in Phase 4 Plan 02 (firmware scaffold)
   - Pin configuration note: same code works on ESP32-S3-BOX-3 and bare DevKitC-1
