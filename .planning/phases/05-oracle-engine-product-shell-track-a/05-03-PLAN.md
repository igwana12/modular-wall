**ENGINE-06: WiFi provisioning via captive portal + ENGINE-04 partial: multi-board config**

1. Create `firmware/oracle-engine/config.h` — shared configuration header:
   - Define `FIRMWARE_VERSION "0.0.1"` (used by OTA module too)
   - Define `DEVICE_NAME "OracleEngine"` (used for AP SSID, mDNS)
   - Define board types: `BOARD_BOX3`, `BOARD_DEVKITC1`
   - Use `#ifndef BOARD_TYPE` / `#define BOARD_TYPE BOARD_DEVKITC1` for default
   - Conditional pin mappings per board type:
     ```
     BOARD_BOX3: I2S_MIC_SCK=41, I2S_MIC_WS=42, I2S_MIC_SD=2, I2S_SPK_BCLK=17, I2S_SPK_LRC=47, I2S_SPK_DOUT=15
     BOARD_DEVKITC1: I2S_MIC_SCK=5, I2S_MIC_WS=6, I2S_MIC_SD=7, I2S_SPK_BCLK=12, I2S_SPK_LRC=13, I2S_SPK_DOUT=14
     ```
   - Note: BOX-3 pins are from its datasheet. DevKitC-1 pins are standard GPIO choices avoiding strapping pins.
   - Define WiFi provisioning AP settings: `AP_SSID_PREFIX "OracleEngine-"`, `AP_PASSWORD ""` (open), `AP_CHANNEL 1`
   - Define server connection defaults: `DEFAULT_SERVER_URL "http://192.168.1.100:8300"`, `DEFAULT_DEITY_ID "apollo"`
   - Define `NVS_NAMESPACE "oracle-eng"` for Preferences storage

2. Create `firmware/oracle-engine/wifi_provision.h`:
   ```cpp
   #ifndef WIFI_PROVISION_H
   #define WIFI_PROVISION_H
   #include <Arduino.h>

   // Initialize WiFi provisioning system. Reads stored credentials from NVS.
   void wifiProvisionInit();

   // Attempt to connect with stored credentials. Returns true if connected within timeoutMs.
   bool wifiProvisionConnect(unsigned long timeoutMs = 15000);

   // Start captive portal for credential entry. Blocks until credentials submitted.
   // Portal runs on 192.168.4.1, redirects all DNS to itself.
   void wifiProvisionPortal();

   // Check if stored credentials exist in NVS.
   bool wifiProvisionHasCredentials();

   // Clear stored credentials (factory reset WiFi).
   void wifiProvisionClear();

   // Get stored server URL (or default).
   String wifiProvisionGetServerUrl();
   ```

3. Create `firmware/oracle-engine/wifi_provision.cpp`:
   - Include WiFi.h, WebServer.h, DNSServer.h, Preferences.h, config.h, wifi_provision.h
   - Static globals: WebServer server(80), DNSServer dnsServer, Preferences prefs, stored ssid/password/serverUrl
   - `wifiProvisionInit()`: Open NVS namespace, read "ssid", "password", "server_url" keys.
   - `wifiProvisionConnect()`: If credentials exist, call WiFi.begin(ssid, password). Poll WiFi.status() in loop with timeout. Return success/failure.
   - `wifiProvisionPortal()`:
     a. Generate AP name: `AP_SSID_PREFIX + last 4 chars of MAC` (unique per device)
     b. Start softAP with generated name
     c. Start DNS server redirecting all domains to 192.168.4.1
     d. Start WebServer with three routes:
        - `GET /` — serve HTML form with fields: WiFi SSID, WiFi Password, Server URL (prefilled with DEFAULT_SERVER_URL), Deity ID (prefilled with DEFAULT_DEITY_ID). Clean, minimal HTML with inline CSS. Title "Oracle Engine Setup".
        - `POST /save` — read form fields, store in NVS via Preferences, send "Saved! Rebooting..." response, delay 1000ms, ESP.restart()
        - `GET /scan` — return JSON array of visible WiFi networks (SSID + RSSI) for the form to populate a dropdown. Call WiFi.scanNetworks().
     e. Loop: dnsServer.processNextRequest() + server.handleClient() + delay(10)
   - `wifiProvisionHasCredentials()`: Check if "ssid" key exists and is non-empty in NVS.
   - `wifiProvisionClear()`: Remove all keys from NVS namespace.
   - `wifiProvisionGetServerUrl()`: Return stored server URL or DEFAULT_SERVER_URL.
   - The HTML form should be a raw string literal (R"rawliteral(...)rawliteral") embedded in the .cpp file. Keep it under 2KB. Include a scan button that fetches /scan and populates SSID dropdown.

4. Update `firmware/oracle-engine/README.md` to add:
   - WiFi provisioning usage: call wifiProvisionInit() in setup(), try wifiProvisionConnect(), fall back to wifiProvisionPortal() if no credentials or connection fails
   - Multi-board compilation: compile with `-DBOARD_TYPE=BOARD_BOX3` for ESP32-S3-BOX-3 or default DevKitC-1
   - All module dependencies: WiFi, WebServer, DNSServer, Preferences, HTTPClient, HTTPUpdate, ArduinoJson (all included with Arduino ESP32 Core 3.x)
