Create two new files and update server.py:

**1. `services/orb-backend/stt.py` — AssemblyAI streaming STT:**
- Use AssemblyAI's REST API (not WebSocket streaming for v1 — simpler, hardware sends complete utterance)
- Function `transcribe_audio(audio_bytes: bytes, sample_rate: int = 16000) -> str` that:
  - Accepts raw PCM 16-bit mono audio bytes
  - Converts to WAV in-memory (add WAV header to raw PCM)
  - Uploads to AssemblyAI via httpx POST to `https://api.assemblyai.com/v2/upload`
  - Creates transcription via POST to `https://api.assemblyai.com/v2/transcript`
  - Polls for completion (GET `/v2/transcript/{id}`) with 500ms interval, 30s timeout
  - Returns transcribed text string
  - Uses `ASSEMBLYAI_API_KEY` env var (already available on Smithers)
- Add `stt_available() -> bool` check function

**2. `services/orb-backend/sphere_ws.py` — WebSocket handler:**
- Import and reuse existing pipeline functions (execute_reading_stream, tts_stream, get_reading_context, load_deity)
- WebSocket protocol (JSON messages):
  - Client sends: `{"type": "config", "deity_id": "apollo", "intent": "guidance on creativity"}`
  - Client sends: `{"type": "audio", "data": "<base64 PCM 16-bit 16kHz mono>"}` (accumulated utterance)
  - Client sends: `{"type": "end_audio"}` (signals end of speech)
  - Server sends: `{"type": "status", "state": "listening|processing|speaking"}`
  - Server sends: `{"type": "transcript", "text": "..."}`
  - Server sends: `{"type": "text", "chunk": "..."}` (LLM tokens)
  - Server sends: `{"type": "audio", "data": "<base64 PCM audio>", "sample_rate": 16000}`
  - Server sends: `{"type": "done"}`
  - Server sends: `{"type": "error", "message": "..."}`
- Handler function `async def sphere_websocket(websocket: WebSocket)`:
  - Accept connection, send `{"type": "status", "state": "ready"}`
  - Wait for "config" message to set deity_id and intent (default deity: apollo)
  - Accumulate "audio" messages into a bytearray (base64 decode each chunk)
  - On "end_audio": run full pipeline:
    1. Send status "processing"
    2. Call `transcribe_audio()` with accumulated PCM bytes
    3. Send transcript back to client
    4. Get RAG context via `get_reading_context(deity_id, transcript)`
    5. Stream reading via `execute_reading_stream(deity_config, transcript, rag_context)`
    6. For each sentence: call `tts_stream()` but request PCM format
    7. Send audio chunks back as base64 PCM
    8. Send "done" when complete
  - Handle WebSocketDisconnect gracefully
  - Log all state transitions

**3. Update TTS for PCM output:**
- Add a new function `tts_stream_pcm(voice_id: str, text: str) -> AsyncGenerator[str, None]` in tts.py
- Same as tts_stream but uses output_format `pcm_16000` instead of `mp3_44100_128`
- ElevenLabs supports `pcm_16000` format natively (16-bit 16kHz mono PCM)
- Yields base64-encoded raw PCM chunks
