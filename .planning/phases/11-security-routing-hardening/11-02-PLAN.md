# Plan 11-02: Context Profiles for Smithers Routing

## Objective
Add context profiles to Smithers routing_policy.json that map task domains (JARVIS, firmware, oracle-cards, etc.) to directory include/exclude patterns, ensuring Claude Code sessions automatically scope to relevant files.

## Context
- @/Volumes/Extreme Pro/ACTIVE/smithers/data/routing_policy.json
- @/Volumes/Extreme Pro/ACTIVE/smithers/router.py
- @/Volumes/Extreme Pro/ACTIVE/smithers/models.py

## Tasks

### Task 1: Add context_profiles to routing_policy.json
type="auto"
Add a context_profiles section with 5 profiles (jarvis, firmware, oracle-cards, infrastructure, kickstarter), each with triggers, include, exclude, and rationale.

### Task 2: Add ContextProfile model and resolve function
type="auto"
Add ContextProfile Pydantic model to models.py. Add resolve_context_profile() to router.py that matches task text against profile triggers.

### Task 3: Wire context_profile into RoutingPlan
type="auto"
Add context_profile field to RoutingPlan model. Pass resolved profile through all build_routing_plan return paths.

### Task 4: Verify task-based routing
type="auto"
Write and run test assertions verifying JARVIS tasks include services/jarvis/ and exclude firmware/, and firmware tasks include firmware/ and exclude services/jarvis/.

## Verification
- JARVIS task routing plan includes services/jarvis/ and excludes firmware/
- Firmware task routing plan includes firmware/ and excludes services/jarvis/
- Generic tasks return no context profile

## Success Criteria
1. A JARVIS task session automatically includes services/jarvis/ directories and excludes firmware/
2. An Orb firmware task session automatically includes firmware/ and excludes services/jarvis/
3. Context profiles are resolved from routing_policy.json configuration
