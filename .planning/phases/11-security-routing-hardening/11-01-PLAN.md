**Architecture:** Keep the existing stdin parsing and tool_name/tool_input extraction. After the existing injection pattern check, add 3 new rule blocks that can each independently block the tool call. If any blocking rule triggers, output `hookSpecificOutput.decision = "block"` with a `reason` string. If no blocking rule triggers, fall through to the existing injection advisory logic.

**Rule 1: Slack Channel Allowlist (per D-04, D-05, D-06)**

```javascript
const SLACK_ALLOWED_CHANNELS = {
  names: ['pantheon', 'the-orb', 'operations'],
  ids: []  // Executor: resolve IDs by grepping Slack config or leave empty — name matching is primary
};

const SLACK_SEND_TOOLS = [
  'mcp__slack__send_message',
  'mcp__slack__post_message', 
  'mcp__slack__chat_postMessage',
  'mcp__slack__reply_to_thread'
];
```

Detection logic:
- If `toolName` starts with `mcp__slack__` AND is in `SLACK_SEND_TOOLS`
- Extract channel from `tool_input.channel` or `tool_input.channel_id` or `tool_input.channel_name`
- If channel is NOT in `SLACK_ALLOWED_CHANNELS.names` (case-insensitive, strip leading #) AND NOT in `SLACK_ALLOWED_CHANNELS.ids`
- BLOCK with reason: `"Slack channel '${channel}' is not in the allowlist. Allowed: #pantheon, #the-orb, #operations. Override by approving the tool call in Claude Code's permission prompt."`

**Rule 2: Git Push Main Protection (per D-07, D-08, D-09)**

Detection logic:
- If `toolName === 'Bash'`
- Extract command from `tool_input.command`
- If command matches regex: `/git\s+push\b.*\b(main|master)\b/`
- BLOCK with reason: `"git push to main/master is blocked. Push to a feature branch instead, or override by approving the tool call."`

**Rule 3: Secret File Protection (per D-10, D-11, D-12)**

```javascript
const SECRET_FILE_PATTERNS = [
  /^\.env$/,
  /^\.env\..+$/,
  /^credentials\.json$/,
  /^secrets\..+$/,
  /\.pem$/,
  /\.key$/,
  /_SECRET/,
  /_TOKEN/,
  /_API_KEY/
];
```

Detection logic:
- If `toolName === 'Write'` or `toolName === 'Edit'`
- Extract basename from `tool_input.file_path` using `path.basename()`
- If basename matches ANY pattern in `SECRET_FILE_PATTERNS`
- BLOCK with reason: `"Writing to secret/credential file '${basename}' is blocked. Override by approving the tool call."`

**Important implementation details:**
- The 3 blocking rules MUST be checked BEFORE the existing injection advisory logic
- If a blocking rule triggers, output the block decision and exit immediately — do NOT also run the injection scan
- The existing injection scan logic (lines 44-96 of current file) MUST remain unchanged
- Use `hookSpecificOutput.decision = "block"` for hard blocks (NOT `additionalContext`)
- Keep the `try/catch` around everything with `process.exit(0)` on error (silent fail pattern)
- Update the file header comments to reflect the new rules
