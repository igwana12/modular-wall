  <action>
Create two new modules in the media_director/ package:

**script_planner.py:**
- Define `ScriptSegment` dataclass with fields: `text` (str), `visual_cue` (str — what to show), `module` (str — which JARVIS module: media/map/chart/banner/effect/solar/etc), `data_query` (dict — params for that module's data fetch), `offset_s` (float — computed later from TTS duration).
- Define `ScriptPlan` dataclass with: `segments` (list[ScriptSegment]), `raw_response` (str — full text for TTS), `planning_model` (str — which Claude model was used).
- Implement `async def plan_script(user_query: str, system_prompt: str, anthropic_client, stage_hints: dict | None = None) -> ScriptPlan`:
  - Calls Claude (claude-haiku-4-5) with a system prompt instructing it to return a JSON array of segments. Each segment has: `text` (the spoken words for that segment), `visual_cue` (what should be shown), `module` (which JARVIS module), `data_query` (params).
  - The system prompt should include: "You are a storyboard planner. Given a user query, generate a response broken into 3-7 segments. Each segment has spoken text AND a visual direction. Return valid JSON array."
  - Parse the JSON from Claude's response. If Claude returns plain text (no JSON), fall back to a single-segment plan with the full text and no visuals (graceful degradation).
  - Concatenate all segment texts to build `raw_response`.
  - Do NOT compute offset_s here — that happens after TTS duration is known.

**parallel_fetcher.py:**
- Define `ResolvedSegment` dataclass: same as ScriptSegment plus `resolved_data` (dict — fetched media results: image URLs, map data, chart data, etc).
- Define `ResolvedStoryboard` dataclass: `segments` (list[ResolvedSegment]), `raw_response` (str).
- Implement `async def fetch_all_media(plan: ScriptPlan, image_matcher=None) -> ResolvedStoryboard`:
  - For each segment with a data_query, create an async task to resolve it. Use the existing `tool_call_to_ws_message()` from tool_definitions.py to convert each segment's (module, data_query) into resolved WS messages.
  - For image queries specifically, use `SemanticImageMatcher.search()` (from semantic_image_matcher.py).
  - Run all fetch tasks via `asyncio.gather(*tasks)` — true parallel execution.
  - Return ResolvedStoryboard with all media pre-loaded.
- Implement `def assign_offsets(storyboard: ResolvedStoryboard, total_duration_s: float) -> ResolvedStoryboard`:
  - Distribute offset_s across segments proportional to text length. Segment N starts at: `sum(len(seg.text) for seg in segments[:N]) / total_chars * total_duration_s`.
  - Mutates offset_s on each segment in place.

Both modules import from existing media_director package — use the same patterns as media_director.py (dataclasses, async/await, type hints).

**TTS duration measurement (ORCH-04):**
- In `_tts_elevenlabs()`, after receiving each MP3 chunk (`r.content`), compute duration from the MP3 data. Use `len(audio_data) * 8 / 128000` as a fast bitrate-based estimate (ElevenLabs Flash v2.5 outputs ~128kbps MP3). Accumulate total across chunks.
- After all chunks sent, send a NEW WebSocket message: `{"type": "tts_duration", "duration_s": total_duration_s}`. This replaces the 0.4s/word estimate with actual measured duration.
- Modify `_tts_elevenlabs` to RETURN the computed `total_duration_s` (float). Currently it returns None. Add the return value without breaking existing callers (they ignore return values already).

**New orchestration function:**
- Add `async def _orchestrate_with_storyboard(user_query, system_prompt, ws, voice_key, stage_hints=None)`:
  1. Call `plan_script(user_query, system_prompt, anthropic_client)` to get ScriptPlan.
  2. In parallel: call `fetch_all_media(script_plan)` AND start TTS of `script_plan.raw_response`.
  3. After TTS completes, use the returned actual duration to `assign_offsets(storyboard, actual_duration_s)`.
  4. Send the storyboard to frontend: `{"type": "storyboard", "segments": [{"offset_s": N, "module": "...", "data": {...}, "text": "..."}]}`.
  5. Falls back to existing `_detect_media_for_response()` if plan_script fails.

**Integration point:**
- In MediaDirector.direct(), add a new optional parameter `script_plan: ScriptPlan | None = None`. When provided, skip the Claude tool-use call and instead convert the pre-planned segments directly into DirectorResult messages. This allows gradual migration — callers that pass a script_plan get the new flow, others get the existing flow.

**Do NOT remove** `_detect_media_for_response()` or the existing MediaDirector flow — they remain as fallback.
