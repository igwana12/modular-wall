# Phase 07.3 Plan 01: Script Planner + Parallel Fetcher + TTS Duration Summary

Pre-planned storyboard architecture with Claude-generated structured JSON scripts, parallel asyncio.gather media fetch, and actual MP3 duration measurement replacing the 0.4s/word estimate.

## What Was Built

### script_planner.py
- `ScriptSegment` dataclass: text, visual_cue, module, data_query, offset_s
- `ScriptPlan` dataclass: segments list, raw_response (concatenated text for TTS), planning_model
- `plan_script()`: async function calling Claude haiku-4-5 with a storyboard planner system prompt. Returns structured JSON array of 3-7 segments, each with spoken text and visual direction. Graceful degradation: returns empty plan if JSON parsing fails (caller falls back to legacy flow).
- Robust JSON parsing handles markdown code fences, extra text, and malformed responses.

### parallel_fetcher.py
- `ResolvedSegment` dataclass: extends ScriptSegment with resolved_data (fetched media results)
- `ResolvedStoryboard` dataclass: list of ResolvedSegments + raw_response
- `fetch_all_media()`: fires all segment data queries via asyncio.gather for true parallel execution. Uses tool_call_to_ws_message() for module resolution and SemanticImageMatcher for image lookups.
- `assign_offsets()`: distributes offset_s proportionally across segments based on text length and actual TTS duration. Updates both segment offset_s and nested WS message offset_s values.

### server.py Modifications
- `_tts_elevenlabs()` now tracks total_audio_bytes, computes actual duration via `bytes * 8 / 128000`, sends `{"type": "tts_duration", "duration_s": N}` to frontend, and returns the float duration (previously returned None).
- `_orchestrate_with_storyboard()`: new async function chaining plan_script -> parallel(fetch_all_media, _tts_elevenlabs) -> assign_offsets -> storyboard WS message. Falls back to legacy flow on any failure.
- Imports added for script_planner and parallel_fetcher modules.

### media_director.py Modifications
- `MediaDirector.direct()` accepts optional `script_plan: ScriptPlan` parameter. When provided, skips Claude tool-use call and converts pre-planned segments directly into DirectorResult messages. Enables gradual migration -- callers without script_plan get the existing flow.

## Deviations from Plan

None -- plan executed exactly as written.

## Decisions Made

1. **MP3 bitrate-based duration**: Used `len(audio_data) * 8 / 128000` for ElevenLabs Flash v2.5 (~128kbps output). Fast computation, no codec dependency. Accurate enough for storyboard timing (within ~5% of actual playback duration).
2. **Gradual migration via optional parameter**: script_plan parameter on MediaDirector.direct() allows both old and new flows to coexist. No existing callers break.
3. **Empty plan = fallback**: If plan_script returns no segments (Claude error, JSON parse failure), _orchestrate_with_storyboard returns False and caller uses legacy _detect_media_for_response.
