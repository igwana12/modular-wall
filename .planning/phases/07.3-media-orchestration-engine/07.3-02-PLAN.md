  <action>
Create a new JS module that manages timed dispatch of storyboard segments synced to audio playback.

**StoryboardScheduler class:**

```
constructor(visualStageController)
```
- Stores reference to VisualStageController for message dispatch.
- Internal state: `_segments` (array), `_startTime` (null until playback starts), `_timers` (array of setTimeout IDs), `_actualDuration` (null until tts_duration arrives).

```
load(storyboardMsg)
```
- Receives the `storyboard` message from WebSocket.
- Stores segments sorted by offset_s.
- Does NOT start timers yet — waits for playback start signal.

```
start()
```
- Called when the first `audio_chunk` message arrives (speech playback begins).
- Records `_startTime = performance.now()`.
- For each segment, creates a `setTimeout` that fires at `segment.offset_s * 1000` ms from now.
- When a timer fires, calls `this._visualStage.handleMessage(segment.data)` — this routes through the existing VisualStageController which already handles media_show, map_show, chart_show, etc.
- If `_actualDuration` is already known (tts_duration arrived before start), rescale all offsets proportionally: `adjusted_offset = segment.offset_s * (_actualDuration / estimated_duration)`.

```
updateDuration(actualDurationS)
```
- Called when `tts_duration` message arrives from backend.
- Stores `_actualDuration`.
- If timers are already running, cancel all pending timers and reschedule with adjusted offsets based on actual duration vs estimated. Adjustment formula: `new_offset = original_offset * (actualDuration / estimatedDuration)` where estimatedDuration was implied by the last segment's offset_s + ~3s buffer.

```
stop()
```
- Clears all pending timers.
- Resets internal state.
- Called on new user input or conversation turn.

```
isActive()
```
- Returns true if a storyboard is loaded and timers are running.

**Edge cases to handle:**
- Storyboard with 0 segments (no visuals) — just play audio, no scheduling.
- tts_duration arrives BEFORE first audio_chunk — store duration, apply when start() is called.
- User interrupts mid-playback (new voice input) — stop() cancels all timers.
- Segment with no data (text-only segment) — skip visual dispatch, just log.

Export as `window.StoryboardScheduler = StoryboardScheduler` (matches existing pattern in this codebase — no ES modules, globals on window).
