## Decisions Made
1. **setTimeout over requestAnimationFrame**: Storyboard events fire at second-scale intervals (not every frame), so setTimeout is more appropriate and CPU-friendly.
2. **Early return for storyboard messages**: storyboard and tts_duration messages return from the WS handler immediately to prevent them from being processed by VisualStageController as unknown message types.
3. **No visual-stage.js changes needed**: handleMessage is already public and exposed as window.jarvisStage -- no modifications required.
