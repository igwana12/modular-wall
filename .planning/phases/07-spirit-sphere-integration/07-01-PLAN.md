**Step 2: Create audio_task.h/cpp** -- Core 0 FreeRTOS wrapper:

audio_task.h:
- `void audioTask(void* param);` -- FreeRTOS task function signature
- `extern TaskHandle_t audioTaskHandle;`

audio_task.cpp:
- #include config.h, audio_i2s.h, ws_client.h, wifi_manager.h, wifi_provision.h, ota_update.h, state_machine.h, mute_button.h, status_display.h
- Implements audioTask():
  1. Call audio_init()
  2. Call wifiProvisionInit() then wifiProvisionConnect() -- uses captive portal from Phase 5
  3. Extract server URL from wifiProvisionGetServerUrl(), parse host/port, call ws_connect()
  4. Call otaInit(serverUrl, FIRMWARE_VERSION)
  5. Declare int16_t audioBuffer[AUDIO_BUFFER_SIZE]
  6. Track SphereState localState = STATE_IDLE
  7. Main loop (while true):
     a. If mute_is_muted() and localState == STATE_LISTENING, skip audio capture
     b. Call ws_loop() for WebSocket event processing
     c. State machine logic (adapted from oracle-engine.ino loop):
        - STATE_READY: wait for PTT press (digitalRead(PTT_BUTTON)==LOW with debounce)
        - STATE_LISTENING: audio_read(audioBuffer, AUDIO_BUFFER_SIZE), ws_send_audio(audioBuffer, AUDIO_BUFFER_SIZE)
        - STATE_SPEAKING: audio_write from ring buffer (same logic as oracle-engine)
     d. On state transition: call state_send(EVT_STATE_CHANGE, newState) and display_update_state(newState)
     e. Call otaCheckForUpdate() (rate-limited internally)
     f. vTaskDelay(pdMS_TO_TICKS(1)) -- yield to WiFi stack
  8. Track free heap every HEARTBEAT_INTERVAL_MS, print to Serial if below 50KB (pitfall 3 monitoring)

  1. Call led_init(), hall_init(), frame_init(), motor_init()
  2. Load deity frame data: frame_load(FRAME_DATA, FRAME_DATA_LEN) -- guarded by #ifdef FRAME_DATA, falls back to test pattern if not defined
  3. Call motor_ramp(TARGET_RPM, MOTOR_RAMP_MS)
  4. Track: uint16_t lastColumn = 0; bool animationActive = false;
  5. Main loop (while true):
     a. Check state queue: while (state_receive(&evt)): if EVT_STATE_CHANGE and data==STATE_SPEAKING -> animationActive=true; if data==STATE_READY or STATE_IDLE -> animationActive=false
     b. Call motor_update() for non-blocking ramp logic
     c. Get period: uint32_t period = hall_get_period_us()
     d. If period==0 or period>HALL_TIMEOUT_US: led_clear(), vTaskDelay(pdMS_TO_TICKS(10)), continue
     e. If animationActive: render POV column (same logic as pov-globe.ino main loop -- get position, map to column, led_show_column)
     f. Else: render idle pattern (slow color cycle or static low-brightness glow)
     g. Column timing delay: uint32_t columnDelay = period / COLUMNS_EFFECTIVE; if (columnDelay > 50) delayMicroseconds(columnDelay - 50);

**Step 4: Create spirit-sphere.ino** -- main sketch:

spirit-sphere.ino:
- #include config.h, state_machine.h, mute_button.h, status_display.h, audio_task.h, led_task.h
- setup():
  1. Serial.begin(SERIAL_BAUD)
  2. Print "Spirit Sphere v1.0.0 booting..."
  3. Call state_init() -- creates FreeRTOS queue
  4. Call mute_init() -- GPIO mute button
  5. Call display_init() -- status display
  6. Create Core 0 task: xTaskCreatePinnedToCore(audioTask, "AudioTask", AUDIO_TASK_STACK, NULL, AUDIO_TASK_PRIORITY, &audioTaskHandle, 0)
  7. Create Core 1 task: xTaskCreatePinnedToCore(ledTask, "LEDTask", LED_TASK_STACK, NULL, LED_TASK_PRIORITY, &ledTaskHandle, 1)
  8. Print "Tasks created. Core 0: Audio, Core 1: LEDs"
- loop():
  1. Handle serial debug commands (deity:<name>, intent:<text>, status, mute)
  2. Print heartbeat every HEARTBEAT_INTERVAL_MS: free heap, WiFi status, sphere state, mute state
  3. vTaskDelay(pdMS_TO_TICKS(100)) -- loop is low-priority monitoring only

IMPORTANT: Use vTaskDelay() everywhere, NEVER delay(). WiFi stack runs on Core 0 -- audio task priority 2 is above WiFi's default 1 to prevent audio glitches (per research pitfall 2). LED task priority 3 is highest -- POV timing is critical.
