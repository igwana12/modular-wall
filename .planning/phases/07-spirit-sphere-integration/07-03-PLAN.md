**tests/07-integration-test-plan.md** -- Step-by-step test procedures:

## Test 1: Firmware Flash and Boot
- Prerequisites: ESP32-S3-BOX-3, USB-C cable, Arduino IDE 2.x with ESP32 Core 3.x
- Steps:
  1. Open firmware/spirit-sphere/spirit-sphere.ino in Arduino IDE
  2. Select board: ESP32-S3-BOX-3
  3. Compile (Verify) -- PASS if 0 errors, 0 warnings about I2S conflicts
  4. Flash to device
  5. Open Serial Monitor at 115200 baud
  6. Expected output within 3 seconds: "Spirit Sphere v1.0.0 booting...", "Tasks created. Core 0: Audio, Core 1: LEDs"
- Pass criteria: Boot message appears, no crash/reboot loop

## Test 2: WiFi Provisioning
- Prerequisites: Test 1 passed, smartphone with browser
- Steps:
  1. On first boot (no stored credentials), device creates AP "SpiritSphere-XXXX"
  2. Connect phone to this AP
  3. Captive portal should auto-open (or navigate to 192.168.4.1)
  4. Enter WiFi SSID, password, and server URL (http://SMITHERS_IP:8300)
  5. Device connects to WiFi -- Serial shows "WiFi connected" with IP address
  6. Device reconnects automatically on next reboot
- Pass criteria: WiFi connected, IP assigned, credentials persisted across reboot

## Test 3: Audio Round-Trip (Voice AI)
- Prerequisites: Test 2 passed, orb-backend running at :8300
- Steps:
  1. Serial Monitor should show "WebSocket connected to /ws/sphere"
  2. Press and hold BOOT button (GPIO 0) -- state changes to LISTENING
  3. Speak a question into BOX-3 microphone
  4. Release BOOT button -- state changes to PROCESSING
  5. Wait for response -- state changes to SPEAKING
  6. Hear deity voice response through BOX-3 speaker
  7. Check latency report in Serial: "STT: Xms, LLM+TTS: Xms, Total: Xms"
- Pass criteria: Hear deity voice response, total latency under 5000ms
- Failure modes: No WebSocket connection (check orb-backend), no audio capture (check I2S config), no response (check STT/LLM/TTS pipeline)

## Test 4: POV LED Display
- Prerequisites: APA102 strip (36 LEDs) connected via slip ring, Hall sensor mounted, N20 motor connected
- Steps:
  1. Motor should start ramping to TARGET_RPM on boot
  2. Serial shows "Motor ramp started" and Hall sensor period values
  3. Verify LED strip shows test pattern (if no image_data.h) or Zeus animation
  4. Observe POV effect: image should be stable, not drifting
  5. Check for LED flicker (slip ring signal quality)
  6. If flicker: reduce SPI_SPEED in config.h (already set to 6MHz, try 4MHz)
- Pass criteria: Stable image displayed, no random color flashes
- Measurement: Use phone slow-motion video (240fps) to verify column alignment

## Test 5: Dual-Core Stress Test
- Prerequisites: Tests 3 and 4 both passing independently
- Steps:
  1. Start POV display (motor spinning, LEDs rendering)
  2. Simultaneously perform voice round-trip (PTT -> speak -> release -> listen)
  3. Observe: audio should not glitch when LEDs refresh
  4. Observe: POV image should not tear or flicker during WebSocket traffic
  5. Run serial command "status" -- verify both tasks report alive
  6. Check free heap: should be above 40KB
  7. Repeat voice round-trip 5 times in succession
- Pass criteria: No audio glitches, no LED artifacts, heap stable above 40KB
- Failure modes: Audio pops (WiFi preempting audio -- check task priorities), LED tear (SPI timing -- check DMA), heap decline (memory leak -- check JSON allocation)

## Test 6: Mute Button
- Prerequisites: Mute button wired to GPIO 39, LED to GPIO 40
- Steps:
  1. Press mute button -- LED should light up, Serial shows "[STATUS] MUTED"
  2. Try PTT while muted -- audio should NOT be captured or streamed
  3. Press mute button again -- LED off, Serial shows "[STATUS] UNMUTED"
  4. Try PTT again -- normal voice round-trip should work
- Pass criteria: Mute toggles correctly, no audio leaks while muted

## Test 7: 10-Minute Demo Stability (SPHERE-07 capstone)
- Prerequisites: All tests 1-6 passing
- Steps:
  1. Start a timer
  2. Run POV display continuously
  3. Perform voice round-trip every 60 seconds (10 conversations total)
  4. Monitor Serial for: crashes, reboots, heap warnings, WebSocket disconnects
  5. Record in bench test log: start time, each voice round-trip result, any anomalies
  6. At 10 minutes: verify POV still running, heap above 40KB, no accumulated errors
- Pass criteria: 0 crashes, 0 reboots, 0 audio dropouts, heap never below 40KB, all 10 voice round-trips succeed
- Record results in tests/07-bench-test-log.md

## Test 8: OTA Update
- Prerequisites: Test 2 passed, new firmware binary available on orb-backend
- Steps:
  1. Place updated firmware binary in services/orb-backend/firmware_bin/
  2. Update manifest.json with new version number
  3. Device should detect update on next OTA check interval
  4. Serial shows "OTA update available: vX.X.X"
  5. Update downloads and installs, device reboots
  6. After reboot, Serial shows new version string
- Pass criteria: OTA update completes, device boots with new version

**firmware/spirit-sphere/spirit-sphere.ino** -- Add to setup() after task creation:

```cpp
#include "esp_task_wdt.h"

// In setup(), after xTaskCreatePinnedToCore calls:
// Initialize Task Watchdog Timer -- 30 second timeout
// If either task hangs for 30s, ESP32 reboots automatically
esp_task_wdt_config_t wdt_config = {
    .timeout_ms = 30000,
    .idle_core_mask = 0,        // Don't watch idle tasks
    .trigger_panic = true       // Reboot on timeout
};
esp_task_wdt_init(&wdt_config);
Serial.println("Watchdog timer initialized (30s timeout)");
```

Add to loop() heartbeat section:
```cpp
// Enhanced heartbeat with stability metrics
if (millis() - lastHeartbeat >= HEARTBEAT_INTERVAL_MS) {
    lastHeartbeat = millis();
    Serial.printf("[HEARTBEAT] uptime=%lus heap=%u minfree=%u wifi=%s state=%d muted=%d\n",
        millis() / 1000,
        ESP.getFreeHeap(),
        ESP.getMinFreeHeap(),
        WiFi.isConnected() ? "yes" : "no",
        (int)currentSphereState,
        mute_is_muted() ? 1 : 0
    );
    // Heap warning threshold
    if (ESP.getFreeHeap() < 40000) {
        Serial.println("[WARNING] Free heap below 40KB!");
    }
    if (ESP.getMinFreeHeap() < 30000) {
        Serial.println("[CRITICAL] Min free heap below 30KB -- possible memory leak");
    }
}
```
