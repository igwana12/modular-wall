Create hardware/enclosure-v1.md -- a detailed specification for the v1 prototype enclosure. This is a design document that will guide CAD modeling in FreeCAD or Fusion 360. The actual CAD files are created during physical prototyping.

**Enclosure Design Philosophy:**
- v1 is a bench prototype, not a production enclosure
- Priority: functional mounting, easy access to components, stable rotation axis
- Material: PLA (cheap, fast to print, adequate for prototype)
- Expect 4-6 print iterations to get fit right

**Two-Part Design:**

### Part 1: Base Enclosure
- Shape: Cylindrical, approximately 130mm diameter x 80mm height
- Houses: ESP32-S3-BOX-3, 3x 18650 battery pack, BMS, buck converter, mute button + LED
- Bottom: Flat with 4 rubber damper feet (vibration isolation from motor per research pitfall 6)
- Top: Center hole (22mm) for slip ring shaft, motor mount platform
- Side openings:
  - USB-C cutout for BOX-3 (programming/charging)
  - USB-C cutout for BMS (battery charging)
  - Mute button mount hole (12mm)
  - Mute LED mount hole (5mm)
  - Ventilation slots (motor heat + ESP32 heat)
- Interior features:
  - BOX-3 cradle/shelf (95x90mm footprint, raised to keep clear of battery)
  - Battery holder mounting points (65mm x 55mm footprint for 3-cell holder)
  - BMS and buck converter mounting posts (M3 standoffs or snap-fit)
  - Wire routing channels
- Wall thickness: 2.5mm (strong enough for PLA prototype)
- Print orientation: upright (base on build plate)

### Part 2: Sphere Mount / Rotating Assembly
- Shape: Vertical arm(s) extending from slip ring rotor
- Attaches to: Slip ring rotating side (rotor)
- Carries: APA102 LED strip (36 LEDs, ~250mm)
- Design options (start with option A):
  - Option A: Single straight arm (simplest, matches Phase 6 flat POV propeller)
  - Option B: Curved arm forming half-sphere arc (better image, harder to balance)
- Arm width: 10mm (enough for 7mm LED strip + adhesive backing)
- Arm length: 250mm (36 LEDs at ~7mm pitch)
- Balance: Counterweight on opposite side if single arm (bolt + nuts)
- Magnet mount: Small pocket at arm tip or base for Hall sensor magnet (3mm disc)

Create hardware/battery-system.md -- comprehensive battery wiring and safety guide.

**Battery System Overview:**
- Configuration: 3S1P (3 cells in series, 1 parallel string)
- Nominal voltage: 11.1V (3 x 3.7V)
- Voltage range: 9.0V (empty) to 12.6V (full)
- Capacity: 2500-3000mAh depending on cell choice (Samsung 25R = 2500mAh)
- Estimated runtime: 2-3 hours (ESP32 ~150mA + LEDs ~500mA average + motor ~200mA = ~850mA total)

**Components:**
1. 3x 18650 Li-ion cells (matched capacity, same brand/model)
2. 3S BMS board with USB-C input (handles balance charging, overcharge, overdischarge, short circuit)
3. LM2596 adjustable buck converter (input: 9-12.6V, output: 5V, 3A)
4. Optional: USB-C PD trigger board (for 12V PD charging -- advanced, defer to v1.1)

**Wiring Diagram (ASCII):**
```
[Cell 1] -- [Cell 2] -- [Cell 3]
  +    -  +    -  +    -
  |    |  |    |  |    |
  B+   B1 B1   B2 B2   B-
  |    |       |       |
  +----[3S BMS Board]--+
       |           |
       P+          P-  ← BMS output (protected, 9.0-12.6V)
       |           |
  [USB-C IN]   [LM2596 Buck]
  (charging)       |      |
                  5V+    GND
                   |      |
            +------+------+------+
            |      |             |
         [ESP32]  [APA102]  [N20 Motor]
         via USB  via slip   via MOSFET
         or 5V    ring
         pin
```

**Safety Requirements (CRITICAL):**
1. NEVER connect 3S pack directly to ESP32 or LEDs (max 5V devices on a 9-12.6V rail = instant destruction)
2. ALWAYS use matched cells (same brand, model, capacity, age) -- mismatched cells cause dangerous imbalance
3. NEVER charge without BMS -- overcharge above 4.2V/cell risks thermal runaway
4. ALWAYS check BMS protection: overcharge (4.2V/cell), overdischarge (2.7V/cell), short circuit (10A+ cutoff)
5. NEVER solder directly to 18650 cells -- use a spot welder or cell holders with spring contacts
6. Keep battery away from heat sources (motor, buck converter)
7. Add a main power switch between BMS output and buck converter (allows full shutdown)

**USB-C Charging Pass-Through:**
- BMS board's USB-C input handles charging (typically 2A @ 5V = 10W)
- For v1: charge and operate simultaneously via USB-C (pass-through)
- Charging time estimate: 2500mAh / 1A = ~2.5 hours (BMS limits charge current)
- Device continues to operate from BMS output while charging

**Buck Converter Setup:**
- LM2596 adjustable module: turn potentiometer to set 5V output
- MEASURE with multimeter before connecting to ESP32 (verify 5.0V +/- 0.1V)
- Input: BMS output (9.0-12.6V)
- Output: 5V rail (powers ESP32, APA102 LEDs via slip ring, motor via MOSFET)
- Maximum load: 3A (ESP32 ~150mA + 36 LEDs at max ~720mA + motor ~200mA = ~1070mA typical, well within 3A)

**Power Budget Table:**

| Component | Voltage | Current (typical) | Current (peak) | Notes |
|-----------|---------|-------------------|----------------|-------|
| ESP32-S3-BOX-3 | 5V | 150mA | 350mA | WiFi + audio active |
| APA102 x36 LEDs | 5V | 360mA | 720mA | 36 x 20mA per LED (RGB all on) |
| N20 Motor | 5V | 100mA | 300mA | Under load (spinning arm + slip ring) |
| Hall sensor | 3.3V | 5mA | 5mA | From ESP32 3.3V regulator |
| Mute LED | 3.3V | 10mA | 10mA | Via 220 ohm resistor |
| BMS quiescent | - | 20mA | 20mA | Standby draw |
| Buck converter | - | 50mA | 50mA | Conversion overhead |
| **Total** | | **695mA** | **1455mA** | |

**Runtime estimate:** 2500mAh / 695mA = ~3.6 hours typical, ~1.7 hours at peak

**Testing Procedure:**
1. Assemble battery pack in holder (DO NOT solder to cells)
2. Connect BMS board to cell holder wires (observe polarity markings: B+, B1, B2, B-)
3. Connect USB-C charger to BMS -- verify charging LED lights
4. Measure BMS output voltage with multimeter: should read 9.0-12.6V depending on charge
5. Connect buck converter input to BMS output
6. Adjust buck converter potentiometer until output reads 5.0V on multimeter
7. Connect ESP32-S3-BOX-3 to 5V rail -- verify boot
8. Connect remaining peripherals one at a time, checking voltage stability
9. Full load test: all peripherals running, verify 5V rail stays above 4.8V
10. Charging test: plug in USB-C while running, verify device continues operating

**References:**
- See hardware/bom.md for component sourcing
- See hardware/wiring-diagram.md for full pin-to-pin connections
- Research pitfall 5: "3S Battery Voltage vs ESP32 Input" -- details voltage regulation chain
