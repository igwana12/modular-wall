    **Step 1: Spherical column mapping in pov-globe.ino.**
    Spherical POV rotates a vertical arc around a vertical axis. The key difference from flat:
    - In flat mode, columns map linearly to angular position
    - In sphere mode, columns still map to angular position BUT the LED positions along the arm represent latitude (pole at LED 0, equator at LED 17, opposite pole at LED 35 for a full-arc arm)
    - The equirectangular image is already mapped correctly by the Python tool (longitude=column, latitude=LED) so the main loop logic stays the same
    - Add config.h: #define POV_MODE_SPHERE true (switch from FLAT)
    - The critical change is TIMING: spherical geometry means LEDs near the poles sweep less angular distance per revolution. The image-to-pov.py tool handles this in the equirectangular mapping, so firmware just needs correct column timing.

    **Step 2: Brightness optimization for ambient visibility (POV-04).**
    POV displays are dim because each LED column is only visible for (column_time / revolution_time) fraction of each revolution. At 3-5 RPM this is a slow sweep, not traditional POV (which needs 300+ RPM for persistence of vision). At this speed the arm is visible as a moving bar of light painting an image over each revolution.

    Strategy for ambient visibility:
    - Set BRIGHTNESS to 255 (maximum APA102 output)
    - APA102 has separate global brightness (5-bit, 0-31) AND per-LED PWM (8-bit per channel). Use global brightness at max (31) for maximum photon output.
    - Add a "boost mode" that doubles the time each column is shown (show each column for 2x normal duration, reducing effective resolution from 120 to 60 columns but doubling perceived brightness)

    Update led_driver.cpp:
    - Set FastLED.setBrightness(255) for maximum output
    - Add led_set_global_brightness() to allow runtime dimming if too bright
    - Add boost mode support: when enabled, show each column twice before advancing

    Update config.h:
    - BRIGHTNESS 255 (maximum for ambient visibility)
    - Add BOOST_MODE false (double-time column display)
    - Add COLUMNS_EFFECTIVE (NUM_COLUMNS if no boost, NUM_COLUMNS/2 if boost)

    **Step 3: Motor soft-start for noise (POV-05).**
    - Verify motor_ramp() uses gradual PWM increase (already in Plan 01 scaffold)
    - Add config.h: MOTOR_RAMP_MS 3000 (3 second ramp to reduce startup noise)
    - Add config.h: MOTOR_HOLD_DUTY — the minimum duty that keeps motor spinning (prevents stall/restart noise). Comment: "CALIBRATE: find lowest duty where motor maintains rotation"

    **Step 4: Add Serial command for live tuning.**
    - '+'/'-' keys: adjust brightness in steps of 25
    - 'f'/'s' keys: faster/slower motor in steps of duty 5
    - 'c' key: print current config (brightness, duty, RPM, column timing)
    - 'p' key: pause/resume column display (motor keeps spinning, LEDs off — for noise measurement per POV-05)
  </action>
