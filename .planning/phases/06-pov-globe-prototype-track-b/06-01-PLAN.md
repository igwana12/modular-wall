  <action>
    Create firmware/pov-globe/ directory with full Arduino project structure.

    **config.h** — Central pin and parameter definitions:
    - APA102 SPI pins: DATA_PIN 11, CLOCK_PIN 12 (ESP32-S3 HSPI defaults)
    - NUM_LEDS 36 (half-sphere arc, 144 LEDs/m density on ~25cm arm)
    - NUM_COLUMNS 120 (angular resolution: 360/120 = 3 degrees per column)
    - SPI_SPEED 12000000 (12MHz, well within APA102 spec of 20MHz+)
    - HALL_PIN 4 (GPIO4 for US5881LUA Hall effect sensor, INPUT_PULLUP)
    - MOTOR_PIN 5 (GPIO5 for N20 motor PWM)
    - MOTOR_PWM_FREQ 1000 (1kHz PWM for quiet operation)
    - MOTOR_PWM_CHANNEL 0
    - MOTOR_PWM_RESOLUTION 8 (0-255 duty cycle)
    - TARGET_RPM 4 (middle of 3-5 RPM range)
    - BRIGHTNESS 200 (out of 255, tunable for ambient visibility per POV-04)
    - Define FASTLED_ESP32_I2S true and FASTLED_ESP32_I2S_NUM_DMA_BUFFERS 4 (per CLAUDE.md)

    **led_driver.h/.cpp** — FastLED APA102 wrapper:
    - led_init(): Call FastLED.addLeds<APA102, DATA_PIN, CLOCK_PIN, BGR>(leds, NUM_LEDS). Set brightness. Use CRGB array.
    - led_show_column(uint16_t col_index, const CRGB* column_data): Copy column_data into LED array and call FastLED.show(). This is the hot path called ~120 times per revolution.
    - led_clear(): Fill black, show.
    - led_set_brightness(uint8_t b): Runtime brightness adjustment for POV-04 ambient tuning.

    **hall_sensor.h/.cpp** — Revolution position tracking:
    - Volatile uint32_t last_trigger_us (set in ISR via micros())
    - Volatile uint32_t revolution_period_us (time between consecutive triggers)
    - Volatile bool new_revolution flag
    - hall_init(): pinMode(HALL_PIN, INPUT_PULLUP), attachInterrupt on FALLING edge (US5881LUA pulls low when magnet detected)
    - ISR (IRAM_ATTR): Record micros(), compute period, set flag
    - hall_get_period_us(): Return revolution_period_us (used to compute column timing)
    - hall_get_position(): Return float 0.0-1.0 representing angular position based on (micros() - last_trigger_us) / revolution_period_us. This is the core sync mechanism for POV-02.
    - hall_new_revolution(): Return and clear new_revolution flag

    **frame_buffer.h/.cpp** — Image data storage:
    - Static CRGB frame[NUM_COLUMNS][NUM_LEDS] array (120 columns x 36 LEDs = 4320 pixels x 3 bytes = ~13KB, fits in ESP32-S3 PSRAM)
    - frame_init(): Zero the buffer
    - frame_load(const uint8_t* data, size_t len): Load raw RGB data into frame array. Data format: sequential columns, each column is NUM_LEDS x 3 bytes (R,G,B).
    - frame_get_column(uint16_t col): Return pointer to frame[col] (zero-copy for performance)
    - frame_load_test_pattern(): Generate a simple test pattern (alternating red/blue hemispheres) for testing without the Python tool

    **motor_control.h/.cpp** — N20 motor PWM:
    - motor_init(): ledcAttach(MOTOR_PIN, MOTOR_PWM_FREQ, MOTOR_PWM_RESOLUTION) (Arduino Core 3.x API, no channel parameter)
    - motor_set_rpm(float rpm): Map rpm to PWM duty cycle. Note: actual RPM mapping requires calibration with real motor. For now, linear map 0-5 RPM to duty 0-180 (conservative, motor-dependent). Add comment: "CALIBRATE WITH REAL MOTOR"
    - motor_ramp(float target_rpm, uint16_t ramp_ms): Soft start/stop over ramp_ms milliseconds to reduce mechanical noise (per POV-05). Use non-blocking millis() approach.
    - motor_stop(): Ramp to 0, then set duty 0.
    - motor_get_duty(): Return current duty for debugging.

    **pov-globe.ino** — Main sketch:
    - setup(): Serial.begin(115200), led_init(), hall_init(), frame_init(), motor_init(). Load test pattern via frame_load_test_pattern(). Print "POV Globe ready" to Serial.
    - loop(): Core POV logic:
      1. If hall_new_revolution(), reset column counter to 0
      2. Compute current column from hall_get_position() * NUM_COLUMNS
      3. Call led_show_column(col, frame_get_column(col))
      4. delayMicroseconds(column_delay) where column_delay = hall_get_period_us() / NUM_COLUMNS
      5. If no hall signal detected (period > 2000000us = <0.5 RPM), call led_clear() as safety
    - Add Serial commands for debugging: 'r' = print RPM, 'b' = cycle brightness, 'm' = toggle motor

    Use Arduino ESP32 Core 3.x API throughout (ledcAttach not ledcSetup). Include guards (#pragma once) on all headers.

  <action>
    Create scripts/image-to-pov.py — a Python CLI tool that converts a PNG/JPG image into POV frame data.

    **Input:** Image file (any size, any aspect ratio)
    **Output:** C header file with const uint8_t array containing column-ordered RGB data

    **Processing pipeline:**
    1. Load image with Pillow (PIL)
    2. For spherical POV: the input image is an equirectangular projection (like a world map). Convert to polar columns:
       - For each of NUM_COLUMNS (120) angular positions (0-360 degrees):
         - Sample a vertical strip from the equirectangular image at the corresponding longitude
         - Resample that strip to NUM_LEDS (36) pixels (the LEDs along the arm)
       - This maps longitude to column index and latitude to LED index
    3. For flat 2D POV (--mode flat): simpler — just resize image to NUM_COLUMNS x NUM_LEDS and read columns left-to-right
    4. Apply gamma correction (gamma 2.2) for LED brightness perception
    5. Output as C header:
       ```c
       // Generated by image-to-pov.py from {filename}
       // {NUM_COLUMNS} columns x {NUM_LEDS} LEDs x 3 bytes (RGB)
       #pragma once
       #include <stdint.h>
       const uint8_t FRAME_DATA[] PROGMEM = {
         // Column 0
         0xRR, 0xGG, 0xBB, // LED 0
         ...
       };
       const size_t FRAME_DATA_LEN = sizeof(FRAME_DATA);
       ```

    **CLI interface (argparse):**
    - `python scripts/image-to-pov.py input.png -o firmware/pov-globe/image_data.h`
    - `--mode flat|sphere` (default: sphere)
    - `--columns 120` (default from config, overridable)
    - `--leds 36` (default from config, overridable)
    - `--gamma 2.2` (default 2.2)
    - `--brightness 1.0` (0.0-1.0 multiplier, for pre-dimming)
    - `--preview` flag: save a preview PNG showing what the POV display will render (columns as vertical strips)

    **Preview mode:** When --preview is set, save a {input}_preview.png that visualizes the frame buffer as a 2D image (NUM_COLUMNS wide x NUM_LEDS tall). Useful for debugging without hardware.

    Add shebang line and make executable. Include docstring with usage examples.
    Require: Pillow, numpy. Add a requirements check at top that prints install instructions if missing.
