    Human assembles hardware and verifies flat POV propeller works. Claude assists with firmware adjustments if calibration is needed.

    **What was built:** Firmware for a flat 2D POV propeller with APA102/SK9822 LEDs, Hall effect position sync, and N20 motor control. All code is ready to flash.

    **Pre-flight (static tests before spinning):**
    1. Flash firmware/pov-globe/ to ESP32-S3 via Arduino IDE
    2. Open Serial Monitor at 115200 baud — confirm "POV Globe ready" message
    3. Press 'l' in Serial — all 36 LEDs should light white (verifies LED wiring)
    4. Wave a magnet past the Hall sensor — Serial should print "Hall trigger" with timing (verifies sensor wiring)
    5. Press 'm' in Serial — motor should spin (verifies motor wiring)

    **POV test (spinning):**
    6. Start motor with 'm', let it reach stable RPM
    7. Serial should show RPM reading between 3-5 (if not, adjust MOTOR duty in config.h)
    8. Observe LED arm while spinning:
       - Should see a circular image (the test pattern) floating in air
       - Image should be STABLE (not drifting) — proves Hall sync works (POV-02)
       - If image drifts: Hall sensor magnet alignment issue, adjust position
       - If image is dim: increase BRIGHTNESS in config.h
       - If columns are misaligned: adjust COLUMN_DELAY_OFFSET_US

    **Calibration (if needed):**
    - If RPM is wrong: adjust motor duty mapping in motor_control.cpp
    - If image appears stretched/compressed: adjust NUM_COLUMNS
    - If colors are wrong: check BGR vs RGB in FastLED.addLeds call
  </action>
