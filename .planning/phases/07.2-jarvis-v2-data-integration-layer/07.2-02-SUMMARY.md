## Decisions Made
- **Bootstrap fix approach:** Direct DB insert of `instance_admin` role instead of running the `auth-bootstrap-ceo` CLI (which requires a running CLI tool not installed on host). The invite flow exists but is overkill for a single-user local deployment.
- **Auth method:** Better Auth email/password sign-in with session cookies. The API also supports agent API keys, but session auth is simpler for a server-side parser.
- **Briefing injection:** Paperclip data is appended to the RSS data block (not a separate source) because the Sonnet synthesis prompt already handles structured text blocks. Adding a fourth source would require prompt changes.
- **Credentials:** Stored in environment variables (PAPERCLIP_EMAIL, PAPERCLIP_PASSWORD) with sensible defaults for local deployment.
