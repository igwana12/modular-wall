## Decisions Made
- Adapter registry uses dict[str, Callable] rather than base class inheritance -- simpler, no ABC overhead for what are essentially handler functions
- RSS cache TTL set to 300s (5 min) separate from main 60s pipeline cache -- feeds update every 15-60 min, sub-minute freshness is unnecessary
- Twitter content auto-detected via multi-signal heuristic requiring 2+ signals (handles density, hashtags, thread markers, twitter URLs, twitter-specific terms)
- Fixed _fetch_resourcerer argument order to match the uniform handler signature (detection, user_text, smithers_result) -- was previously swapped
