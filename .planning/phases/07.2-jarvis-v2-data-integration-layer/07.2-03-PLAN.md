**1. Add widget_show WebSocket event emission (server.py):**

Find where DataPayload results are consumed after `DataPipeline.fetch()` returns (in the Smithers/data pipeline handler section). Currently, data flows into the LLM system prompt via `_build_user_message_with_data()` — the LLM speaks it but the frontend never sees the structured data.

Add widget_show emission alongside the existing data injection:

```python
# After DataPipeline returns a payload:
if payload and payload.items:
    # Send structured data to widget panel for visual display
    await ws.send_json({
        "type": "widget_show",
        "data": payload.model_dump()
    })
```

This means: when live data has structured items (RSS articles, tweets, papers, tasks), show them visually AND speak about them. When data is chart-only (candlestick, line chart), continue using chart_show as before.

Decision logic:
- `payload.items` non-empty -> `widget_show` (card display)
- `payload.chart_type` set and `payload.data` has labels/values -> `chart_show` (chart display)
- Both present -> send both events (cards + chart side by side, max-2 panel rule handles it)

```python
# In PerplexityParser.parse():

items = []

    # ... existing fields
)
```

The key insight: Perplexity already returns citations and snippets — the parser just needs to structure them as items instead of dumping them into the flat `data` dict.
