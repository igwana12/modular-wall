    **1. lib/db.ts** -- Lightweight user storage:

    For v1, use a JSON file as user store (no database dependency yet). This keeps the phase focused on the payment flow, not database setup. Store at `data/users.json` (gitignored).

    ```typescript
    interface User {
      id: string;
      email: string;
      tier: 'free' | 'premium';
      stripeCustomerId: string | null;
      stripeSubscriptionId: string | null;
      createdAt: string;
    }

    export async function getUserByEmail(email: string): Promise<User | null>;
    export async function createUser(email: string): Promise<User>;
    export async function updateUserTier(email: string, tier: 'free' | 'premium', stripeCustomerId?: string, stripeSubscriptionId?: string): Promise<void>;
    export async function getUserByStripeCustomerId(customerId: string): Promise<User | null>;
    ```

    Implementation: read/write `data/users.json` with fs. Use a simple array of User objects. Add `data/` to .gitignore.

    **2. lib/auth.ts** -- Auth.js v5 config:

    - Use Resend provider for magic link email (per RESEARCH.md)
    - JWT session strategy (no database sessions needed)
    - Add `AUTH_SECRET` env var (generate with `npx auth secret`)
    - Provider config: `Resend({ from: process.env.EMAIL_FROM })` -- falls back to console.log in dev if AUTH_RESEND_KEY not set
    - Callbacks: `jwt` callback stores user email + tier in token; `session` callback exposes tier to client
    - `signIn` callback: creates user in db.ts if not exists

    Export `{ handlers, signIn, signOut, auth }` from NextAuth(config).

    **3. app/api/auth/[...nextauth]/route.ts**:

    ```typescript
    import Stripe from 'stripe';

      apiVersion: '2024-12-18.acacia', // or latest stable
    });
    ```

    **5. app/api/checkout/route.ts**:

    - POST handler, requires authenticated session (from auth())
    - Creates Stripe Checkout session:
      - `mode: 'subscription'`
      - `line_items: [{ price: process.env.STRIPE_PRICE_ID, quantity: 1 }]`
      - `success_url: ${origin}/account?session_id={CHECKOUT_SESSION_ID}`
      - `cancel_url: ${origin}/pricing`
      - `customer_email: session.user.email` (pre-fill from auth)
    - Returns `{ url: session.url }` for client redirect

    **6. app/api/webhooks/stripe/route.ts**:

    - POST handler, NO auth (Stripe calls this directly)
    - CRITICAL: Use `req.text()` not `req.json()` for raw body (per RESEARCH.md Pitfall 3)
    - Verify signature: `stripe.webhooks.constructEvent(body, sig, process.env.STRIPE_WEBHOOK_SECRET!)`
    - Handle events:
      - `checkout.session.completed`: extract customer email, set tier='premium', store stripeCustomerId and subscriptionId
      - `customer.subscription.deleted`: find user by stripeCustomerId, set tier='free'
      - `customer.subscription.updated`: update subscription status
      - `invoice.payment_failed`: log warning (future: notify user)
    - Return 200 for all events (Stripe retries on non-200)

    **7. app/api/billing-portal/route.ts**:

    - POST handler, requires authenticated session
    - Finds user's stripeCustomerId from db
    - Creates Stripe Billing Portal session: `stripe.billingPortal.sessions.create({ customer: stripeCustomerId, return_url: ${origin}/account })`
    - Returns `{ url: session.url }`
