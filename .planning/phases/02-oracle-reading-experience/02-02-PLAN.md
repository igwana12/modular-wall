    **1. reading-stream.tsx** (`'use client'`)

    Props: `{ deityId: string; intent: string; onComplete: () => void; }`

    State management:
    - `readingText: string` -- accumulated full reading text
    - `deityImage: DeityEvent['image']` -- from deity SSE event
    - `ttsError: boolean` -- true if tts_error received
    - `fatalError: string | null` -- from error event

    Uses `useOracleSSE` hook:
    - URL: `/api/oracle/read/${deityId}?intent=${encodeURIComponent(intent)}`
    - onText: append chunk to readingText state
    - onAudio: call `audioQueue.enqueue(chunk)`
    - onDeity: store image for display
    - onDone: call `onComplete()`
    - onError: set fatalError
    - onTtsError: set ttsError=true

    Uses `useAudioQueue` hook for audio playback

    Rendering (per UI-SPEC Reading Stream States):
    - Connecting: shadcn Skeleton pulse in reading area + "The oracle stirs..." (Label size, muted text)
    - Streaming: Text rendered in a `<div>` with `max-w-[640px]` (optimal reading measure per UI-SPEC), Body typography (16px/1.5/Inter). Each new text chunk appended. Token fade: `opacity: 0 -> 1` via CSS transition `duration: 0.15` (per UI-SPEC -- CSS, not motion, for perf)
    - PANTHEON art: `<Image>` of deity image rendered after first paragraph break or mid-reading, `rounded-xl`, full width, lazy loaded
    - Audio playing indicator: rendered via AudioPlayer component positioned top-right
    - TTS error: small warning badge "Voice unavailable -- your reading continues as text." (per UI-SPEC)
    - Fatal error: red-tinted card border, "The oracle cannot speak right now. The connection to Olympus has been disrupted. Try again in a moment." (per UI-SPEC)
    - Complete: "Your reading is complete." divider, then two buttons side-by-side: "Pull Another Card" (link to `/`) and "View Guidebook" (link to `/guidebook/${deityId}`)
    - `aria-live="polite"` on text container (per UI-SPEC accessibility)

    **2. BFF proxy routes** -- thin proxies to orb-backend:

    `src/app/api/deities/[deity_id]/route.ts`:
    - GET handler, fetches `${process.env.ORB_BACKEND_URL}/api/deities/${deity_id}`
    - Returns JSON response, caches with `next: { revalidate: 3600 }`
    - 404 if backend returns 404

    `src/app/api/content/[deity_id]/random/route.ts`:
    - GET handler, fetches `${process.env.ORB_BACKEND_URL}/api/content/${deity_id}/random`
    - Returns JSON response
    - `export const dynamic = 'force-dynamic';` (random = no cache)

    **3. /read/[deity_id]/page.tsx** -- Server Component orchestrator:

    - This is a SERVER component that fetches deity config and a random PANTHEON image at request time
    - Fetches deity config: `fetch(\`${process.env.ORB_BACKEND_URL}/api/deities/${deity_id}\`, { next: { revalidate: 3600 } })`
    - Fetches random image: `fetch(\`${process.env.ORB_BACKEND_URL}/api/content/${deity_id}/random\`)`
    - If deity not found, call `notFound()`
    - Renders a CLIENT wrapper component (ReadingPageClient) that manages the reading flow state machine:
      - Phase 1: CardReveal (shows card flip, passes deity data + image URL)
      - Phase 2: IntentInput (slides up after reveal completes)
      - Phase 3: ReadingStream (starts after intent submitted)
    - The ReadingPageClient manages state: `phase: 'reveal' | 'intent' | 'reading' | 'complete'`
    - AudioQueue.initAudio() called on "Begin Reading" button tap (user gesture for AudioContext)
    - Back arrow in top-left (link to `/`), Mute icon in top-right (AudioPlayer)
    - All wrapped in DeityBackground component with colorPalette from deity config
    - Layout: content centered, max-width per UI-SPEC reading page layout, padding 16px mobile / 32px tablet+

    Per READ-01: SSR the card back for fast FCP, lazy-load motion for card flip via `next/dynamic({ ssr: false })`.
