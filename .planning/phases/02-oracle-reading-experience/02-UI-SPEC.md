# Phase 2 -- UI Design Contract

> Visual and interaction contract for the Oracle Reading Experience. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (init at project scaffold time via `npx shadcn@latest init`) |
| Preset | Default dark -- customize after init: `cssVariables: true`, `style: "new-york"`, `baseColor: "zinc"` |
| Component library | Radix UI (ships with shadcn) |
| Icon library | Lucide React (ships with shadcn) |
| Font | `Inter` for UI text, `Crimson Pro` for deity names and reading headings (serif for mythological tone) |
| Animation library | motion 12.x (import from `motion/react`) |
| Theme | next-themes with `defaultTheme: "dark"`, `attribute: "class"` |

**Source:** CONTEXT.md (locked: shadcn/ui, next-themes dark default), RESEARCH.md (motion 12.38.0).

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, input internal padding |
| md | 16px | Default element spacing, card padding, form gaps |
| lg | 24px | Section padding within reading flow |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page section breaks (card reveal to intent input) |
| 3xl | 64px | Top/bottom page padding on mobile |

Exceptions:
- Touch targets: minimum 44px height/width for all interactive elements (mobile-first requirement)
- Card dimensions: 280px wide x 420px tall (tarot ratio 2.75:4.75, from RESEARCH.md card-reveal.tsx)

## Typography

| Role | Size | Weight | Line Height | Font Family | Usage |
|------|------|--------|-------------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Inter | Reading text, descriptions, guidebook prose |
| Label | 14px | 500 (medium) | 1.4 | Inter | Form labels, metadata, tier badges, secondary info |
| Heading | 24px | 600 (semibold) | 1.2 | Crimson Pro | Section titles, deity title below card, guidebook headings |
| Display | 32px | 700 (bold) | 1.1 | Crimson Pro | Deity name on card reveal, homepage hero text |

**Rationale:** Two font families create a clear split between UI chrome (Inter, clean/modern) and mythological content (Crimson Pro, serif/classical). Both are Google Fonts, free, well-hinted for mobile screens.

**Loading strategy:** `next/font/google` with `display: 'swap'`. Crimson Pro subset to Latin only (mythology content is English). Inter is the shadcn default -- already optimized.

### Per-Deity Dynamic Color

Each deity has a `color_palette` array of 3 hex values in their JSON config. These are applied dynamically when a deity is active:

| Palette Index | Application |
|---------------|-------------|
| `[0]` (primary) | Deity name text color, reading container top border/glow |
| `[1]` (secondary) | Background gradient overlay (low opacity, 5-10%), reading accent |
| `[2]` (tertiary) | Subtle UI highlights during reading (progress bar, audio visualizer) |

**Implementation:** CSS custom properties set via `style` attribute on the reading container when the `deity` SSE event arrives. Example: `--deity-primary: #ffd600; --deity-secondary: #87ceeb;`

Light mode is secondary (toggle available, not promoted). Dominant flips to `hsl(0 0% 98%)` / `#fafafa`, secondary to `#ffffff`, border to `#e4e4e7`. Accent gold stays the same. Deity colors remain unchanged. next-themes handles the class toggle.

## Interaction States

### Card Reveal Sequence (the core moment)

| Step | Duration | Visual | Interaction |
|------|----------|--------|-------------|
| 1. Page load | 0ms | Card BACK visible (SSR), dark gradient with sacred geometry | None -- passive |
| 2. Anticipation | 0-500ms | Subtle glow pulse on card back edges (gold accent) | None -- build tension |
| 3. Card flip | 500-1700ms | 3D Y-axis rotation, spring physics (stiffness: 60, damping: 20) | None -- animation plays |
| 4. Art revealed | 1700ms | Deity PANTHEON art fully visible, card settles with micro-bounce | None |
| 5. Deity info | 2000ms | Name (Display size, deity primary color) + title (Heading size, muted) fade in below card | None |
| 6. Background shift | 2000ms | Page background gains subtle gradient overlay using deity secondary color at 8% opacity | None |
| 7. Intent input | 2300ms | Input area slides up from bottom with motion `initial={{ y: 40, opacity: 0 }}` | User can type question |
| 8. Submit | User action | "Begin Reading" button (gold accent) | Tap to start SSE stream |

### Reading Stream States

| State | Visual | Copy |
|-------|--------|------|
| Connecting | Skeleton pulse in reading area, 3 animated dots | "The oracle stirs..." |
| Streaming text | Text appears token-by-token, smooth opacity fade per token | (reading content) |
| Streaming audio | Subtle waveform/pulse indicator near deity image, gold tertiary color | Audio icon animated |
| Audio paused | Waveform static, mute icon visible | "Tap to resume" tooltip |
| Complete | Full text visible, "Reading complete" divider, share/save actions appear | "Your reading is complete." |
| Error (fatal) | Red-tinted card border, error icon | "The oracle cannot speak right now. Try again in a moment." |
| TTS error (non-fatal) | Small warning badge near audio control | "Voice unavailable -- reading continues as text." |

### Paywall States

| State | Visual | Copy |
|-------|--------|------|
| Approaching limit | Subtle counter: "1 free reading remaining this month" | Shown as Label-size text below reading |
| Limit reached | Dialog overlay on next reading attempt | See Copywriting Contract below |
| Authenticated free | Counter persists, server-enforced | "Sign in to keep your reading history" |
| Premium active | No counter, no gate, premium badge in nav | "Premium" badge, gold accent |

### Daily Card States

| State | Visual | Copy |
|-------|--------|------|
| Not pulled today | Large CTA button, gold accent, centered on homepage | "Pull Your Daily Card" |
| Already pulled | Smaller deity card thumbnail + "Today's card: {deity name}" | "You drew {Deity Name} today. Read again?" |
| Loading pull | Button shows spinner, disabled | "The gods are choosing..." |

## Animation Contract

All animations use the `motion` library (12.x) imported from `motion/react`.

| Animation | Trigger | Properties | Duration/Physics |
|-----------|---------|------------|------------------|
| Card flip | Page load + 500ms delay | `rotateY: 0 -> 180` | `type: 'spring', stiffness: 60, damping: 20` (~1.2s) |
| Deity info fade | Card flip complete | `opacity: 0 -> 1, y: 20 -> 0` | `duration: 0.4, delay: 0.3` |
| Intent input slide | Deity info complete | `opacity: 0 -> 1, y: 40 -> 0` | `type: 'spring', stiffness: 100, damping: 25` |
| Reading text token | SSE text event | `opacity: 0 -> 1` per token span | `duration: 0.15` (CSS transition, not motion -- perf) |
| Background color shift | deity SSE event | CSS custom property transition | `transition: background 1.5s ease` (CSS only) |
| Gallery tile hover/tap | User interaction | `scale: 1 -> 1.03` | `type: 'spring', stiffness: 300, damping: 20` |
| Paywall dialog | Limit reached | `opacity: 0 -> 1, scale: 0.95 -> 1` | `duration: 0.2` |
| Audio pulse | Audio playing | `scale: 1 -> 1.1 -> 1` loop | `duration: 1.5, repeat: Infinity, ease: 'easeInOut'` |
| Page transition | Route change | `opacity: 0 -> 1` | `duration: 0.2` |

**Performance rule:** Only the card reveal uses motion spring physics. All other animations use simple CSS transitions or lightweight motion tweens. No layout-triggering animations (no width/height/margin animations).

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Color contrast | All text meets WCAG AA (4.5:1 body, 3:1 large text) against dominant background |
| Deity color contrast | Deity primary colors that fail contrast on dark bg get `text-shadow` glow treatment |
| Focus indicators | Visible focus ring on all interactive elements (shadcn default: ring-2 ring-ring) |
| Reduced motion | `@media (prefers-reduced-motion)`: skip card flip (show front immediately), disable audio pulse, disable token fade |
| Screen reader | Card reveal: `aria-live="polite"` on deity name region. Reading stream: `aria-live="polite"` on text container |
| Audio controls | Visible mute/unmute button. Audio never autoplays without user gesture (AudioContext created on "Begin Reading" tap) |
| Touch targets | Minimum 44x44px for all buttons and interactive elements |
| Skip to content | Hidden skip link to reading content area |
