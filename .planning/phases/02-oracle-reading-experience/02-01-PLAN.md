  <action>
    1. Create `src/types/sse-events.ts` -- typed interfaces matching the exact SSE event shapes from orb-backend streaming.py:

    ```typescript
    export interface DeityEvent {
      name: string;
      title: string;
      image: { path: string; tags: string[] } | null;
      color_palette: [string, string, string];
    }
    export interface TextEvent { chunk: string; }
    export interface AudioEvent { chunk: string; } // base64-encoded
    export interface DoneEvent { total_chars: number; deity: string; }
    export interface ErrorEvent { error: string; }
    export interface TtsErrorEvent { error: string; }

    export type SSEStatus = 'idle' | 'connecting' | 'streaming' | 'done' | 'error';
    ```

    2. Create `src/types/deity.ts` -- typed deity config matching gods/*.json shape:

    ```typescript
    export interface DeityConfig {
      id: string;
      name: string;
      title: string;
      voice_id: string;
      system_prompt: string;
      mythology_keywords: string[];
      reading_style: string;
      art_collection: string;
      color_palette: [string, string, string];
      mckee_guidance: string;
    }

    export interface DeitySummary {
      id: string;
      name: string;
      title: string;
      color_palette: [string, string, string];
    }
    ```

    3. Create `src/app/api/oracle/read/[deity_id]/route.ts` -- BFF SSE proxy:

    - Export `const runtime = 'edge';` for Vercel streaming (no timeout limit)
    - Export `const dynamic = 'force-dynamic';` to prevent caching
    - GET handler reads `deity_id` from `params` (await params as Next.js 15 requires)
    - Reads `intent` from searchParams
    - Validates intent length >= 3, returns 400 if too short
    - Fetches `${process.env.ORB_BACKEND_URL}/api/oracle/read/${deity_id}?intent=${encodeURIComponent(intent)}` with `Accept: text/event-stream` header
    - Returns `new Response(backendResponse.body, { headers: { 'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache', 'Connection': 'keep-alive' } })`
    - If backend returns non-200, forward the error status

    4. Create `src/hooks/use-sse.ts` -- the `useOracleSSE` hook:

    - Takes `url: string | null` and `callbacks: { onDeity?, onText?, onAudio?, onDone?, onError?, onTtsError? }`
    - Returns `{ status: SSEStatus }`
    - Uses `useEffect` keyed on `url` -- when url is non-null, creates `new EventSource(url)`
    - CRITICAL: Uses `es.addEventListener('deity', ...)` NOT `es.onmessage` -- orb-backend emits named events
    - Each listener parses `JSON.parse(e.data)` and calls the corresponding callback
    - Sets status to 'connecting' on open, 'streaming' on first deity event, 'done' on done event, 'error' on error
    - Cleanup: `es.close()` in useEffect cleanup
    - Also listens for `tts_error` event (non-fatal, calls onTtsError)

    5. Create `src/hooks/use-audio-queue.ts` -- Web Audio API queue:

    - Creates AudioContext lazily (on first enqueue call, NOT on mount -- per browser autoplay policy)
    - `initAudio()` function: creates AudioContext, must be called from user gesture (Begin Reading button click)
    - `enqueue(base64Chunk: string)` -- decodes base64 to ArrayBuffer, decodes audio, queues for sequential playback
    - `setMuted(muted: boolean)` -- mutes/unmutes via gain node
    - `isMuted` state exposed
    - `isPlaying` state exposed
    - Sequential playback: source.onended triggers next chunk from queue
    - Returns `{ initAudio, enqueue, setMuted, isMuted, isPlaying }`
