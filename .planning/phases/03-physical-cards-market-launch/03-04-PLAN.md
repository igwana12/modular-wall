**Step 3: Build card front template**
Create `apps/oracle/src/app/print/[deity]/page.tsx`:
- Server component that reads deity config from `services/orb-backend/gods/{deity}.json`
- Also reads master CSV row for supplemental data (sacred_colors, genesis_title)
- Card dimensions: 900x1500px (3.0" x 5.0" at 300 DPI — includes 0.125" bleed)
- Safe area: inner 825x1425px (2.75" x 4.75" trim line)
- Layout:
  - Full-bleed hero image as background (cover, center)
  - Semi-transparent gradient overlay at bottom for text readability
  - Roman numeral (I-XXI) top center, deity color accent
  - Deity name large, deity title smaller below
  - QR code bottom-right, 1.25" minimum (375px), white background padding for scan reliability
  - Sacred Circuits small logo or wordmark bottom-left
  - Use deity color_palette from JSON for accents, text glow, border elements
- The hero image path: check /Volumes/Extreme Pro/MYTHS/{nn}-{DEITY}/images/hero/ first, pick first image found. If deity not in MYTHS numbering (Eros, Pan, Nike, Tyche, Persephone, Hestia), search /Volumes/Extreme Pro/sacred-circuits-outputs/organized/ for matching filenames. As last resort, use a placeholder gradient with deity colors.
- Embed the QR code inline by reading the SVG file from assets/qr-codes/{deity_id}.svg

**Step 6: Build generation script**
Create `scripts/generate-card-designs.ts`:
- Reads all 21 deity IDs from services/orb-backend/gods/ directory
- Maps each deity to its Roman numeral (I-XXI) based on the ordering in the plan:
  I=Zeus, II=Hera, III=Poseidon, IV=Hades, V=Athena, VI=Apollo, VII=Artemis,
  VIII=Ares, IX=Aphrodite, X=Hermes, XI=Hephaestus, XII=Demeter,
  XIII=Dionysus, XIV=Persephone, XV=Hestia, XVI=Prometheus, XVII=Hecate,
  XVIII=Eros, XIX=Pan, XX=Nike, XXI=Tyche
- Starts the Next.js dev server (or uses existing if running)
- For each deity:
  1. Copy best hero image to apps/oracle/public/print-assets/{deity}.png (Playwright needs accessible URL)
  2. Navigate Playwright to http://localhost:3000/print/{deity}
  3. Screenshot at 900x1500 viewport, deviceScaleFactor=2 (1800x3000 actual pixels for print quality)
  4. Save to assets/card-designs/front-{deity_id}.png
- Also screenshot /print/back → assets/card-designs/back.png
- Also screenshot /print/instruction → assets/card-designs/instruction.png
- Print summary of generated files
