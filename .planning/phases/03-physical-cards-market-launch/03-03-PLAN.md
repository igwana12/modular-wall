**Domain Registration:**
1. Go to Cloudflare Registrar (dash.cloudflare.com > Domain Registration > Register Domain)

3. Purchase with 2-year registration ($140 total — .ai requires 2-year minimum)
4. If oracleball.ai is unavailable, backup options: theoracle.ai, oraclecards.ai

**DNS Configuration (after purchase):**
1. In Cloudflare DNS for oracleball.ai, add:
   - A record: `@` -> `76.76.21.21` (Vercel)
   - CNAME record: `www` -> `cname.vercel-dns.com`
2. In Vercel project settings > Domains:
   - Add `oracleball.ai`
   - Add `www.oracleball.ai` with redirect to apex
3. Vercel auto-provisions SSL certificate

**After DNS propagates (usually < 5 minutes):**
Claude will verify the domain resolves correctly.

- Set `NEXT_PUBLIC_APP_URL=https://oracleball.ai` in Vercel environment settings
- This updates the Stripe Checkout success/cancel URLs automatically

<task type="auto">
  <name>Task 2: Kit email integration and deposit-to-tag wiring</name>
  <files>apps/oracle/src/app/api/webhooks/stripe/route.ts, apps/oracle/src/lib/kit.ts</files>
  <read_first>apps/oracle/src/app/api/webhooks/stripe/route.ts, .planning/phases/03-physical-cards-market-launch/03-RESEARCH.md</read_first>
  <action>
1. Create `apps/oracle/src/lib/kit.ts` — Kit (ConvertKit) API client:
   - Kit API v3 base URL: `https://api.convertkit.com/v3/`
   - Export async function `addSubscriber(email: string, tag: string)`:
     - POST to `https://api.convertkit.com/v3/tags/{tag_id}/subscribe`
     - Body: `{ api_secret: process.env.KIT_API_SECRET, email }`
     - Map tag names to IDs using env vars: `KIT_TAG_VIP_DEPOSIT`, `KIT_TAG_EMAIL_ONLY`
   - Export async function `addToForm(email: string)`:
     - POST to `https://api.convertkit.com/v3/forms/{form_id}/subscribe`
     - Body: `{ api_secret: process.env.KIT_API_SECRET, email }`
     - Form ID from env: `KIT_FORM_ID`
   - Handle errors gracefully (log but don't throw — email tagging is non-critical)

2. Extend webhook handler (`apps/oracle/src/app/api/webhooks/stripe/route.ts`):
   - In the `reservation_deposit` branch (added in Plan 01), after logging:
     - Import and call `addSubscriber(email, 'vip-deposit')` to tag the depositor in Kit
     - This triggers the VIP email sequence automatically via Kit automation rules

3. Create email signup endpoint for non-deposit signups:
   - Create `apps/oracle/src/app/api/subscribe/route.ts`
   - POST handler accepts `{ email }`
   - Calls `addToForm(email)` to add subscriber to the main Kit form
   - Returns `{ success: true }`
   - This supports the "Just sign up for updates" secondary CTA on the landing page

4. Update the deposit-cta.tsx component (from Plan 02):
   - The secondary "sign up for updates" button should POST to `/api/subscribe` instead of `/api/deposit`

5. Add env vars needed (document in .env.example or similar):
   - KIT_API_SECRET
   - KIT_FORM_ID
   - KIT_TAG_VIP_DEPOSIT (tag ID for VIP depositors)
   - KIT_TAG_EMAIL_ONLY (tag ID for email-only signups)

Note: The user will need to create the Kit account and get API credentials (Task 1 of Plan 03 or separately). The code should work with or without Kit credentials — if KIT_API_SECRET is not set, skip Kit calls silently.
