# Phase 12 Plan 01: Teleport and Remote Control Configuration

## Objective

Enable cross-device session mobility for Claude Code: start a session on Mac desktop and continue it from iPhone/iPad via SSH tunnel through existing Cloudflare infrastructure. Create a teleport script that packages session context and a remote access endpoint accessible from mobile Safari.

## Context

- @.planning/ROADMAP.md (Phase 12 goals)
- @.planning/REQUIREMENTS.md (MOBIL-01, MOBIL-02, MOBIL-03)
- Claude Code CLI supports `--resume`, `--continue`, `--session-id`, `--name` for session persistence
- Cloudflare Tunnel already configured at `~/.cloudflared/`
- SSH available on the Mac
- No built-in teleport command exists in Claude Code CLI v2.1.91

## Tasks

### Task 1: Create teleport script and session mobility toolkit
type="auto"

Create a shell script that enables session mobility:

1. `teleport.sh` script that:
   - Lists recent Claude Code sessions with `claude --resume` (captures session IDs)
   - Exports current session context (working directory, git branch, recent files, conversation ID)
   - Generates a resume command that can be run on another terminal/device
   - Supports `teleport export` (save state) and `teleport resume <id>` (restore state)
   
2. `remote-claude.sh` script that:
   - Starts Claude Code in `--print` mode listening on stdin/stdout
   - Wraps it with a simple HTTP API using `socat` or `nc` for remote access
   - OR sets up SSH tunnel config for direct terminal access from mobile

3. Cloudflare Tunnel configuration for SSH access:
   - Add SSH service to existing tunnel config
   - Configure `claude.local` or similar hostname for remote SSH
   - Document the mobile workflow: Termius/Blink app -> CF Tunnel -> SSH -> claude --resume

**Done criteria:**
- `teleport.sh export` produces a JSON file with session context
- `teleport.sh resume` reconstructs the session from that context
- Cloudflare tunnel config updated with SSH endpoint
- Remote access documented in a README

### Task 2: Cross-device verification checkpoint
type="checkpoint:human-verify"

User physically tests the session mobility flow:
1. Start a session on desktop Mac
2. Use teleport to export session
3. Connect from phone (via Termius/Blink SSH client through Cloudflare Tunnel)
4. Resume the session on phone
5. Verify conversation context is preserved

**Done criteria:**
- User confirms they can resume a session from their mobile device
- User confirms conversation history is accessible on the remote connection

## Verification

- [ ] teleport.sh script exists and is executable
- [ ] Running `teleport.sh export` produces valid JSON output
- [ ] Cloudflare tunnel config includes SSH service definition
- [ ] README documents the full mobile workflow

## Success Criteria

1. User can export a Claude Code session and resume it on another device
2. Remote SSH access works through Cloudflare Tunnel from a mobile client
3. Session context (conversation history, working directory) is preserved across devices
